# Kwork Sniper

Telegram-бот для мониторинга новых заказов на бирже фриланса [Kwork.ru](https://kwork.ru). Парсит страницу проектов в реальном времени и моментально отправляет уведомления подписчикам с гибкой системой фильтров и AI-оценкой заказов.

## Возможности

- **Мониторинг в реальном времени** — проверка новых заказов каждые 20 секунд
- **Фильтрация по категориям** — Дизайн, Разработка и IT, Тексты и переводы, SEO и трафик, Соцсети и маркетинг, Аудио/видео/съемка, Бизнес и жизнь
- **Фильтр по ключевым словам** — получайте только заказы, содержащие интересующие вас слова в заголовке или описании
- **Фильтр по цене** — настройка минимальной и максимальной стоимости заказа
- **Тихий час** — задайте время, когда уведомления не будут приходить
- **AI-анализ заказов** — автоматическая оценка привлекательности заказа по шкале 1–10 через LLM (Groq / OpenAI-compatible API)
- **Supabase** — хранение пользователей и настроек в облачной БД

## Архитектура

| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа. Объединяет парсер и Telegram-бота в одном asyncio-процессе |
| `parser.py` | Парсинг HTML Kwork, извлечение JSON с заказами, очистка и форматирование данных |
| `telegram_bot.py` | Обработчики команд и inline-кнопок (aiogram 3.x), настройки пользователя |
| `database.py` | Клиент Supabase REST API: пользователи, фильтры, ключевые слова, цены, тихий час, AI-настройки |
| `ai_analyzer.py` | Запросы к LLM для оценки заказов, кэширование результатов |
| `bot.py` | Standalone консольный парсер (без Telegram) для быстрой проверки |

## Установка

```bash
# 1. Клонирование репозитория
git clone <repo-url>
cd KworkSniper

# 2. Установка зависимостей
pip install -r requirements.txt

# 3. Настройка переменных окружения
# Скопируйте .env.example в .env и заполните своими данными:
cp .env.example .env
#
# BOT_TOKEN=your_telegram_bot_token
# SUPABASE_URL=https://your-project.supabase.co
# SUPABASE_KEY=your_supabase_anon_key
# AI_BASE_URL=https://api.groq.com/openai  # или другой OpenAI-compatible endpoint
# AI_TOKEN=your_ai_api_key
# AI_MODEL=openai/gpt-oss-120b
```

## Запуск

### Полная версия (Telegram-бот + парсер)

```bash
python main.py
```

Бот запустится в режиме polling и начнёт мониторинг Kwork.

### Только консольный парсер

```bash
python bot.py
```

Выводит новые заказы в консоль. Настройте `KEYWORDS` внутри файла `bot.py` при необходимости.

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Главное меню, регистрация пользователя |
| `/status` | Текущие настройки фильтров и уведомлений |

## Настройки через интерфейс бота

В главном меню доступны следующие опции:

- **🔔 Уведомления** — включить/выключить рассылку
- **🤖 AI-анализ** — включить оценку заказов нейросетью и задать минимальный порог оценки (1–10)
- **⚙️ Фильтры категорий** — выбрать интересующие категории заказов
- **🔑 Ключевые слова** — задать список слов для фильтрации (через запятую)
- **💰 Диапазон цены** — установить минимальную и максимальную цену заказа
- **🌙 Тихий час** — задать период, когда уведомления не приходят

## AI-оценка заказов

При включённом AI-анализе каждый заказ оценивается по шкале 1–10 на основе:
- Соотношения цены к трудозатратам для разработчика
- Рутинности задачи
- Чёткости технического задания
- Специализации (заказы вне IT оцениваются строже)

Оценки визуально размечаются: 🔥 (7–8) и 🔥🔥 (9–10).

## Требования

- Python 3.10+
- Зависимости: `aiogram>=3.7.0`, `httpx>=0.27.0`, `python-dotenv>=1.0.0`
- Аккаунт Telegram BotFather для создания бота
- Проект Supabase с таблицей `users`
- (Опционально) API-ключ для Groq или другого OpenAI-compatible сервиса

## Структура таблицы users (Supabase)

```sql
create table users (
  id bigint primary key,
  username text,
  notifications_enabled boolean default true,
  filters jsonb default '[]',
  keywords jsonb default '[]',
  min_price integer,
  max_price integer,
  quiet_hours_start integer,
  quiet_hours_end integer,
  ai_enabled boolean default false,
  ai_min_score integer
);
```
